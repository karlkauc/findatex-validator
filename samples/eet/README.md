# EET V1.1.3 example files

Generated by `tools/build_eet_samples.py`. One clean reference plus
seven deliberately broken files exercising the EET rule set.

| File | What it demonstrates |
|------|----------------------|
| `01_clean.xlsx` | All M-flagged fields populated; SFDR Art-8 product layout. |
| `02_missing_mandatory.xlsx` | Blanks NUM=5, 7, 23, 25, 26 → PRESENCE/* errors. |
| `03_bad_formats.xlsx` | Invalid ISO 8601 datetime / date, non-ISO-4217 currency, non-Y/N reporting flag. |
| `04_sfdr_art8_no_min.xlsx` | SFDR Art-8 product without Art-8 minimum proportions → `EET-XF-ART8-MIN-LT`, `EET-XF-ART8-MIN-SI`. |
| `05_sfdr_art9_no_min.xlsx` | SFDR Art-9 product without Art-9 minimum proportions → `EET-XF-ART9-MIN-LT`, `EET-XF-ART9-MIN-ENV`, `EET-XF-ART9-MIN-SOC`. |
| `06_sfdr_out_of_scope_with_art_fields.xlsx` | NUM=27="0" (out-of-scope) but Art-8/Art-9 fields populated → `EET-XF-ART30-MUST-BE-ABSENT`, `EET-XF-ART31-MUST-BE-ABSENT`, … (WARNING). |
| `07_pai_yes_block_missing.xlsx` | NUM=33="Y" (product considers PAI) but the PAI block (NUMs 103/104/106/…/202) is empty → 27× `EET-XF-PAI-*` (ERROR). |
| `08_taxonomy_min_unattributed.xlsx` | NUM=41 reports an Art-8 minimum but NUMs 42/43/44 are blank → `EET-XF-ART8-MIN-SI-SPLIT` (WARNING). |
